Vault Selector

Each case can contain multiple vaults for organising evidence by category. Click the vault name at the top of the left pane to open the vault dropdown. From here you can switch between vaults, search vaults, or create a new one with "+ Add New Vault".

Framework Phases & Smart Folders

The left pane organises documents by framework phase. For RICS, the six phases are: Pre-Contract, Construction, Claims & Variations, Delay & EOT, Financial, and Close-Out. Click a phase to expand its smart folders. Click a folder to see the documents filed within it.

Selecting Evidence for Analysis

In Analysis mode, click documents in the left pane to add them to your analysis context. Selected documents appear under a "USER CONTEXT" section with a count badge. The analysis engine uses these documents as reference material when processing your queries.

4

Uploading & Managing Documents

After case creation, add documents through the File Manager or by dragging files onto the workspace.

Opening the File Manager

Click Open Files in the left pane footer to open the File Manager. You can also access it from the header toolbar. The File Manager provides full control over evidence in the current vault.

Uploading Files

1

Drop files onto the upload zone

Drag files from your desktop onto the dashed drop zone, or click "Select Folder" to browse. Accepted formats: PDF, DOCX, XLSX, MD, TXT, XER, MPP, DWG, PNG, JPG, TIFF, BMP, MSG, EML, PST, OST.

2

Review the file queue

Queued files show their name, size, and a "Ready" status badge. Remove unwanted files with the delete button.

3

Zero Trust validation

Files pass through the validation pipeline: Staging → Validation → Confirmed. A progress bar shows validation status per file.

4

Evidence confirmation

Validated files are promoted to confirmed evidence and appear in the vault. Each receives an Evidence ID (e.g. DOC-001) and initial tags.

Managing Evidence

In the File Manager, use the toolbar to filter by type (PDF, Word, Excel, Programme, Drawing, Image, Email), sort by name/date/size, and filter by validation status (Pending, Confirmed, Rejected). Select files using checkboxes for bulk operations: set validation status, add or remove tags, or delete from the vault.

Running Analysis (Analysis Mode)

Switch to Analysis mode by clicking the A button. The centre pane becomes a conversation panel where you submit queries against your case evidence.

1

Select evidence in the left pane

Click documents in the vault to add them to the analysis context. A "USER CONTEXT" section appears showing selected documents with a count badge.

2

Type or dictate your query

Write your question in the input field, or click the microphone for voice dictation. You can also attach additional files directly to the message.

3

Review the response

The response appears in the conversation with structured findings. Multi-step analyses show expandable workflow steps.

4

Continue the conversation

Ask follow-up questions. The conversation maintains context from previous messages for multi-turn analysis.

Prompt Linter

The Prompt Linter is a real-time quality bar that appears above the input field as you type. It analyses your query and provides instant feedback on quality, completeness, and specificity before you submit.

ElementDescription
ScoreQuality score from 0 to 100. Green (80+) = good, Amber (50–79) = needs improvement, Red (<50) = poor. The entire bar changes colour to match.
LanguageDetected language (EN / FR) based on keyword patterns.
Task TypeInferred task: create, modify, explain, analyze, delay-analysis, rea-preparation, concurrent-analysis, etc.
TokensEstimated token count for the input.
CostEstimated processing cost for the query.
IssuesColoured badges for detected problems: high (missing methodology, conflicts), medium (vague references, ambiguous quantities), low (missing dates, format suggestions).
Auto-fixGradient button that applies the linter's suggested rewrite. Appears only when improvements are available.

Rewrite Preview

When the Linter detects issues or an active Skill offers a tailored rewrite, a side-by-side preview panel appears below the linter bar showing the Original text and the Auto-fixed version. Click Auto-fix to replace your input with the improved version, or ignore it to keep your original text.

When a Skill is active, the linter may show multiple Skill Rewrite options — each tailored to a specific methodology or discipline. Click any rewrite option to apply it.

Linter Score Thresholds The score is computed from word count, presence of technical and domain terms, numbers, date references, file extensions, and methodology keywords. A higher score means a more precise, actionable prompt that will produce better results.

Activating a Skill

1

Open the Skills panel

Click the DA button in the input toolbar. The panel lists all available skills with their descriptions.

2

Select a skill

Click a skill to activate it. The active skill shows an "ok" badge and the DA button tooltip updates to show the active skill name.

3

Submit your query

Type your query as usual. The active skill shapes the analysis approach and response structure to match the selected discipline.

4

Deactivate

Click the active skill again to deactivate it and return to the default analysis mode.

Automatic Skill Detection

The Prompt Linter automatically detects relevant skills based on your input. When it recognises trigger patterns (e.g. "time impact analysis", "Eichleay formula", "adjudication referral"), it suggests the appropriate skill and may offer a rewritten prompt optimised for that discipline. You can accept the suggestion or keep your original text.

Banners & Notifications

THEMiS-QS uses two notification layers to keep you informed: a top banner for system-wide alerts, and a notification stack (top-right) for transient event messages. Both support four severity levels.

Top Banner

The top banner is a fixed bar that spans the full width of the screen. It appears for system-level events such as updates, backend status, and analysis progress. Dismiss it with the × button, or click the action button when one is shown.
